Pytorch还包含很多针对特定领域应用的模块如torchaudio，torchtext，torchvision，**skorch（sklearn+pytorch）**等。

经常使用的三个模块torch.nn（神经网络），torch.optim（优化器）和torch.utils.data（数据集以及数据加载模块）。

经常出现的问题：

  • Tensor on different device to model，将数据和模型都放到GPU上去，使用to方法；
  • Mismatched Dimension
  • Mismatched tensor type
  • Cuda out of memory，数据没法一次性全部放入GPU，减小batch大小

class ExampleDataset(torch.utils.data.Dataset):
    def __init__(self):
    	self.data = "abcdefghijklmnopqrstuvwxyz"
    def __getitem__(self,idx):
        if idx >= len(self.data): # if the index >= 26, return upper case letter
      		return self.data[idx%26].upper()
    	else: # if the index < 26, return lower case, return lower case letter
      		return self.data[idx]

过程中一些值得注意的代码，比如设置随机数、可视化训练过程、设置超参数及模块化实现模型加载训练过程等部分。

# 设置随机数使结果可复现
myseed = 42069  
torch.backends.cudnn.deterministic = True  # 使用确定性卷积
torch.backends.cudnn.benchmark = False  #如果每次训练时计算图不会改变时（每次输入形状相同，模型不改变）设为True可提高性能，反之降低性能
np.random.seed(myseed)
torch.manual_seed(myseed)
if torch.cuda.is_available():
    torch.cuda.manual_seed_all(myseed)

# 以返回属性值+可变长参数传入的方式设置优化器
optimizer = getattr(torch.optim, config['optimizer'])(model.parameters(), **config['optim_hparas'])

加载数据时的“num_worker”设置，数值大则寻batch速度快，因为下一轮迭代的batch很可能在上一轮/上上一轮…迭代时已经加载好了。坏处是内存开销大，也加重了CPU负担。经验设置值是自己CPU数，如果CPU很强、RAM也很充足，就可以设置得更大些。如果设为0，则仅主进程加载，dataloader不再有自主加载数据到RAM这一步骤，而是在RAM中找batch，找不到时再加载相应的batch，速度更慢。
