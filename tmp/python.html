一. Pythonic思考方式

要点一 版本问题

要点二 PEP8风格

8号python增强提案，可以直接在IDE中设置，或者使用Pylint等静态源码分析工具。一些规则如下：

  • 受保护的实例属性用单个下划线开头，私有实例属性用双下划线开头；
  • 类与异常，采用单词字母均大写的方式
  • 类中实例方法，第一个参数为self表示该对象自身
  • 类方法的首个参数，命名为cls，表示类自身
  • 引入模块时一定使用绝对路径，比如from boo import foo
  • import按顺序分为三个部分，标准库模块、第三方模块以及自用模块，每个部分内按字母排序

要点三 字符序列类型

Python2中“str”表示原始的8位值，“unicode”包含Unicode字符；Python3中则是“bytes”表示原始的8位值，“str”包含Unicode字符。

UnionCode和二进制（即原始的8位值）的相互转化使用encode和decode方法，比较常用的是UTF-8。

编写Python程序的时候，在最外围进行编码和解码。

**问题1：**Python2中如果str只包含7位ASCII字符，则str和Unicode成为同一类型3。Python3中绝对不可以等价混用，空字符串也不行。

**问题2：**Open函数获取文件句柄时，Python2中默认使用二进制形式，而Python3默认使用UTF-8编码，如果操作二进制文件需要传入‘wb’参数。

要点四 辅助函数

表达式如果变得复杂，要考虑把它拆解成小块，并把逻辑移入函数中。不要过度使用Python的语法特性，写成难以理解的表达式。

要点五 切割序列

如果列表要保留开头/结尾，不需要再切片中写0或者len(list)，会更清爽。

切片操作不考虑索引是否越界。

对原列表切割后会生成一份新的列表，在新列表上进行修改不会影响原列表。赋值的时候则会直接替换。

作者建议尽量不要在一个切片操作中，同时制定start，end和stride，可以拆分成两步，先做范围分割再处理步长。另外尽量对stride只用正数。（这个就是要点六）

要点七 列表推导式

列表推导式可以根据一个列表比较简单地生成另一个列表。

尽量使用列表推导式替代map和filter。

字典和集合类型也有相同的推导机制。

列表推导式中支持多重循环，但尽量不要超过两层，同层的if条件语句间默认使用and。

为了保证后续阅读难度不要太大，尽量不要编写看似简洁但难以理解的代码。

要点九 生成器表达式

推导式中会对于每一个输入的值都要创建仅包含一项元素的全新列表，只适用于处理少量的输入值，否则会占用太多内存。

生成表达式是对列表推导和生成器的一种泛化，运行时不会把整个输出序列都呈现出来，而是估值为一个迭代器，逐次调用next函数生成新值。生成表达式的格式即将列表表达式的中括号换为圆括号。

value = [len(x) for x in open('/tmp/my_file.txt')]
it = (len(x) for x in open('/tmp/my_file.txt'))
next(it)

生成器还可以相互作用，比如某个生成器所返回的迭代器放在另一个生成器表达式的for子表达式中，串在一起的生成器执行速度很快。

roots = ((x, x**0.5) for x in it)
print(roots)

要点十 使用enumerate

enumerate可以指定索引起始值，默认为0。

要点十一 zip遍历两个列表

Python3中的zip函数可以把两个或两个以上的迭代器封装成生成器，会从每个迭代器中获取下一个值，然后汇聚成元组。

Python2中的zip函数并不是一个生成器而是把所有元素完全汇聚出来一次性返回，如果想达到相同效果可以使用itertools中的izip函数。

如果两个列表中的元素数目不同，只要有一个耗尽，zip就不再产生元组了，提前终止。如果不确定列表是否等长，可以使用itertools中的zip_longest函数。

要点十二 for和while后面跟else

Python有特殊的语法，可以在for，while循环后面加else，如果整个循环体中没有遇到break语句就会执行else，但是尽量不要这样写！

要点十三 try/except/else/finally代码块的利用

try/finally结构比较常见，既要异常向上传播，又要在异常发生时进行清理工作。比如确保程序能可靠地关闭文件句柄，其实with语句有实现这个效果。

try/except/else结构可以清晰描述哪些异常会自行处理，哪些会上报上级。try块中没有发生异常就会执行else块。else块可以缩减try块中的代码量，而且如果发生异常会向上传播。

混合使用完整的try/except/else/finally结构，比如要从文件中读取某项事务的描述信息，处理事务并更新文件。可以用try读取文件内容并处理，用except对应try块中可能发生的相关异常，else事实更新文件并把更新文件异常回报给上级代码，然后用finally（finally块总是会执行）清理文件句柄。

二. 函数

要点十四 函数返回值

None和0以及空字符串之类的值，都会在条件表达式里判定为False，使用None这个返回值表示特殊意义的函数，很容易使调用者犯错。

函数在遇到特殊情况（比如除法不允许被除数是0等）应该抛出异常而不是返回None，并在开发文档中写明。
